# Brag Plan: OpenMemoryUI

## What is this app?
OpenMemoryUI is a transparent agentic memory playground where every chat turn is traced through session, episodic, semantic, and contextual memory.

## Planning rubric
1. What is the app? A browser-only Memory Glassbox that lets you talk to an agent and watch memory receive, retrieve, generate, extract, and write in real time.
2. Funniest or most impressive claim? "Talk to an agent whose memory is a glass box."
3. Visual hook? The dark split-screen UI with four color-coded memory stores and a five-stage pipeline.
4. Actual UI to show? The chat column, pipeline strip, memory panels, contextual score bars, memory bus log, and provenance drawer.
5. Shortest satisfying video? 21.5 seconds: enough for hook, product reveal, one message flow, memory writes, and outro.
6. Tone? Preset: app-store; creative direction: polished product-launch demo for an AI memory debugger.
7. Audio feel? Warm business-y music bed with restrained UI clicks, soft reveal hits, and one final bell.
8. Share caption? OpenMemoryUI makes agent memory inspectable: every turn, retrieval, write, and provenance trail in one glassbox UI.
9. User flow worth showing? Type a personal fact -> pipeline stages light up -> facts and episodes write to memory -> provenance and prompt x-ray explain exactly why.

## The angle
Most AI memory demos ask for trust. This one opens the casing. The video should feel like a serious product launch for people who want to see where every remembered fact came from.

## Hook (first 2-3 seconds)
Open on the product name with four memory-store dots resolving into the line: "Memory isn't magic. It's traceable."

## Key moments (the middle)
- The actual glassbox UI appears: chat on the left, four memory stores on the right.
- A sample message is typed and sent while Receive -> Retrieve -> Generate -> Extract -> Write lights up.
- Session, semantic, episodic, and contextual panels populate with cards, scores, and memory-bus writes.
- The provenance drawer opens to show why a fact was stored and where it came from.

## Outro / punchline
"OpenMemoryUI. Transparent agentic memory, ready to launch."

## User flow worth showing
Type: "My name is Manish and I work as a data engineer." The message lands in session memory, retrieval checks long-term stores, the demo agent replies, extraction finds durable facts, and the UI writes semantic and episodic memories with provenance.

## Tone
- Preset: app-store
- Creative direction: polished product-launch demo for an AI memory debugger
- Interpretation: Clean feature-forward pacing, readable labels, dark product UI, restrained motion, and enough rhythm to feel launch-ready without becoming generic SaaS.

## Audio direction
- Role: warm, polished launch bed with sparse professional accents.
- Music: happy-beats-business-moves-vol-11-by-ende-dot-app.mp3.
- Music treatment: start at 0.0s, volume around 0.34, fade visually through final logo; no voiceover.
- Music cue guidance: bundled preset at assets/music/cues/happy-beats-business-moves-vol-11-by-ende-dot-app.music-cues.json, tempo 114.84 BPM. Major reveal targets near 1.60s, 3.70s, 8.96s, 12.65s, 17.91s. Sequential pipeline lights can use the 8.44-10.54 beat window, but readability wins.
- Audio-reactive treatment: subtle; hero glow and panel presence may breathe lightly with the music. No waveform, equalizer, notes, or strobing.
- SFX posture: sparse; one click for send, soft hits for UI reveals, one final bell.
- Audio-coupled moments: product-name reveal, app UI reveal, send click, pipeline step sequence, memory-card writes, final logo.
- Restraint rule: never let SFX compete with the UI text; no dense typing ticks.
